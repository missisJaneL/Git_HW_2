Структура баг-репорта

1. ID
2. Название бага
3. Короткое описание (Summary)
4. Проект (Project)
5. Компонент приложения (Component)
6. Номер версии (Version)
7. Cерьезность (Severity)
8. Приоритет (Priority)
9. Статус (Status), но многие системы баг трекинга выставляют статус по умолчанию
10. Автор (Author) - устанавливается по умолчанию, если нет, то указывается имя автора баг репорта
11. Назначен на (Assigned To) - менеджер проекта или менеджер, ответственный за разработку компонента, на который написан баг репорт
12. Окружение (ОС / Сервис Пак и т.д. / Браузера + версия / ...)
13. Шаги воспроизведения (Steps to Reproduce)
14. Фактический Результат (Result)
15. Ожидаемый результат (Expected Result)
16. Дополнительные материалы (видео, фото -(Attachment)).
